# 极简部署指南 - 复制粘贴版

> 预计时间：15 分钟 | 需要准备：邮箱地址
> ⚠️ 本指南中的所有命令都可以**直接复制粘贴**，不需要你自己写代码！

---

## 第一步：上传代码到 GitHub（5 分钟）

### 1.1 注册 GitHub

1. 打开 🔗 https://github.com
2. 点击右上角 **Sign up**
3. 输入邮箱 → Continue
4. 创建密码 → Continue
5. 输入用户名（英文，如 `li-ming-2024`）→ Continue
6. 邮箱验证 → 选 **Continue for free**（免费版）

### 1.2 创建仓库

1. 点击右上角 **+** → **New repository**
2. Repository name: `ahst-major`（英文，简单好记）
3. 选择 **Public**
4. ✅ 勾选 **Add a README file**
5. 点击 **Create repository**

### 1.3 上传所有文件

1. 在仓库页面，点击 **Add file** → **Upload files**
2. 打开文件夹 `D:\firstlove\11`
3. **把里面所有文件和文件夹**（frontend、backend、database 等）**拖到网页**
4. 等进度条走完
5. 滚动到页面底部 → 点击绿色按钮 **Commit changes**

✅ 完成后**告诉我你的 GitHub 仓库地址**（类似 `https://github.com/你的用户名/ahst-major`），我帮你检查是否上传成功！

## 第二步：部署后端到 Render（5-10 分钟）

### 2.1 注册 Render

1. 打开 🔗 https://render.com
2. 点击 **Get Started**
3. 选择 **Sign up with GitHub**
4. 用 GitHub 账号登录 → 授权

### 2.2 创建后端服务

1. 点击 **+ New** → **Web Service**
2. 选择 **Public Git repository**
3. 粘贴你的仓库地址：
   ```
   https://github.com/你的用户名/ahst-major
   ```
   （把「你的用户名」替换成实际的用户名）
4. 点击 **Continue**

### 2.3 复制粘贴以下配置

| 项目 | 复制粘贴内容 |
|------|--------------|
| **Name** | `ahst-major-backend` |
| **Environment** | 从下拉框选 `Python` |
| **Branch** | `main`（默认就是这个） |
| **Build Command** | 👇 复制下面这一整段 |

```
pip install --upgrade pip && pip install -r backend/requirements.txt && cd backend && python manage.py migrate --no-input && python manage.py collectstatic --no-input --clear && python ../database/import_data.py && python ../add_2024_2025_scores.py
```

| 项目 | 复制粘贴内容 |
|------|--------------|
| **Start Command** | 👇 复制这一段 |

```
cd backend && gunicorn ahst_project.wsgi:application --bind 0.0.0.0:$PORT --workers 2 --timeout 120
```

| 项目 | 复制粘贴内容 |
|------|--------------|
| **Plan** | 点击选择 **Free**（$0） |

### 2.4 添加环境变量

在页面下方找到 **Environment Variables**，点击 **Add Environment Variable**，分别添加 4 条：

**第 1 条：**
```
Key: DJANGO_SECRET_KEY
Value: （点击旁边的 "Generate" 按钮自动生成）
```

**第 2 条：**
```
Key: DJANGO_DEBUG
Value: False
```

**第 3 条：**
```
Key: USE_SQLITE
Value: True
```

**第 4 条：**
```
Key: DJANGO_LOG_LEVEL
Value: INFO
```

### 2.5 开始部署

1. 点击底部的 **Create Web Service**
2. 耐心等待 3-5 分钟
3. 会看到滚动的日志（黑色文字），只要不是红色错误就正常
4. 成功后，页面顶部会显示类似：
   ```
   https://ahst-major-backend-xxxx.onrender.com
   ```

### 2.6 验证后端

在浏览器地址栏输入（替换为你的实际地址）：

```
https://ahst-major-backend-xxxx.onrender.com/api/statistics/
```

如果看到一串数字和大括号 `{...}`，不是空白或「404 Not Found」，就说明后端成功了！

✅ 现在**告诉我你的后端地址**，我帮你配置下一步！

## 第三步：部署前端到 Vercel（5 分钟）

### 3.1 注册 Vercel

1. 打开 🔗 https://vercel.com
2. 点击 **Sign Up**
3. 选择 **Continue with GitHub**
4. 登录并授权

### 3.2 导入项目

1. 点击 **Add New...** → **Project**
2. 找到你的仓库 `ahst-major`，点击 **Import**

### 3.3 配置项目

| 项目 | 操作 |
|------|------|
| **Project Name** | `ahst-major-recommend`（可默认） |
| **Framework Preset** | 下拉选择 `Vite` |
| **Root Directory** | 点击 **Edit** → 输入 `frontend` → **Continue** |
| **Build Command** | 保持默认 `npm run build` |
| **Output Directory** | 保持默认 `dist` |

### 3.4 添加环境变量（**必须正确，否则推荐功能无法使用**）

在 **Environment Variables** 区域：

```
Name: VITE_API_BASE_URL
Value: https://你的后端地址.onrender.com/api
```

⚠️ **重要：把「你的后端地址」替换成第二步获得的实际地址！**

正确示例：
```
VITE_API_BASE_URL = https://ahst-major-backend-abc123.onrender.com/api
```

点击 **Add** 按钮确认添加。

### 3.5 开始部署

1. 点击 **Deploy**
2. 等待 2-3 分钟
3. 成功后获得类似：
   ```
   https://ahst-major-recommend.vercel.app
   ```

## 常见问题速查

### Q1: Render 部署一直显示 "Building"？
**A**: 正常！第一次部署需要下载安装 Python 依赖，请耐心等待 3-5 分钟。

### Q2: Render 日志出现红色错误？
**A**: 截图发给我，我帮你看具体问题。

### Q3: Vercel 部署成功，但点击「推荐」没反应？
**A**: 99% 是 `VITE_API_BASE_URL` 填错了。检查：
- 有没有拼写错误
- 地址末尾有没有 `/api`
- 后端地址是否正确（先单独在浏览器打开后端地址测试）

### Q4: 提示 "502 Bad Gateway" 或 "404 Not Found"？
**A**: 服务还在启动中，等待 1-2 分钟刷新即可。

### Q5: 免费层能支持多少人使用？
**A**: 几十个同学同时使用完全没问题，平时正常使用不会超限。
