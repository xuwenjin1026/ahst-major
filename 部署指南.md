## 第一步：准备 GitHub 账号

1. 访问 https://github.com
2. 注册一个账号（已有则直接登录）
3. 创建一个新仓库（New Repository）
   - 仓库名称: `ahst-major-recommendation`
   - 类型: Public（公开）
   - 勾选 "Add a README file"
4. 把你的项目文件（整个 `11` 文件夹里的内容）上传到这个 GitHub 仓库

### 快速上传方法

在你的项目目录下，打开 PowerShell/命令行，运行：

```powershell
# 初始化 Git（如果还没有）
git init
git add .
git commit -m "初始提交"

# 连接到 GitHub 仓库（替换为你的用户名和仓库名）
git remote add origin https://github.com/你的用户名/ahst-major-recommendation.git

# 推送代码
git branch -M main
git push -u origin main
```

或者直接使用 GitHub Desktop 拖拽上传。

## 第二步：部署后端到 Render.com（免费）

### 2.1 注册 Render 账号

1. 访问 https://render.com
2. 点击右上角 "Sign Up"
3. 选择 "Sign up with GitHub"（推荐，方便后续一键部署）
4. 登录你的 GitHub 账号并授权

### 2.2 部署后端服务

**方法一：使用 render.yaml 自动配置**

1. 在 Render 仪表盘点击 "New" → "Web Service"
2. 选择 "Public Git repository"
3. 粘贴你的 GitHub 仓库地址
4. 填写以下配置：

```
Name:           ahst-major-backend
Environment:    Python
Branch:         main
Root Directory: backend
Build Command:  ./build.sh
Start Command:  gunicorn ahst_project.wsgi:application --bind 0.0.0.0:$PORT --workers 2 --timeout 120
Plan:           Free（选择免费套餐）
```

5. 在 "Advanced" → "Environment Variables" 添加：

| 变量名 | 值 |
|--------|-----|
| `DJANGO_SECRET_KEY` | 点击 "Generate" 自动生成 |
| `DJANGO_DEBUG` | `False` |
| `USE_SQLITE` | `True` |
| `DJANGO_LOG_LEVEL` | `INFO` |

6. 点击 "Create Web Service"
7. 等待 3-5 分钟部署完成

### 2.3 获取后端 API 地址

部署成功后，Render 会给你一个地址，类似：

```
https://ahst-major-backend-xxxx.onrender.com
```

**把这个地址记下来**，下一步要用到。

### 2.4 验证后端是否正常工作

在浏览器访问：

```
https://你的后端地址.onrender.com/api/statistics/
```

如果看到 JSON 数据，说明后端部署成功！

## 第三步：部署前端到 Vercel（免费）

### 3.1 注册 Vercel 账号

1. 访问 https://vercel.com
2. 点击 "Sign Up"
3. 选择 "Continue with GitHub"
4. 登录并授权你的 GitHub 仓库访问

### 3.2 部署前端项目

1. 在 Vercel 仪表盘点击 "Add New" → "Project"
2. 选择你的 GitHub 仓库 `ahst-major-recommendation`
3. 点击 "Import"
4. 填写配置：

```
Project Name:  ahst-major-recommend
Framework:     Vite
Root Directory: frontend
Build Command: npm run build
Output Directory: dist
```

5. 在 "Environment Variables" 部分，添加一个变量：

```
Name:   VITE_API_BASE_URL
Value:  https://你的后端地址.onrender.com/api
```

⚠️ **重要**：把 `你的后端地址` 替换为第二步中获得的实际地址。

例如：
```
VITE_API_BASE_URL = https://ahst-major-backend-abc123.onrender.com/api
```

6. 点击 "Deploy"
7. 等待 2-3 分钟部署完成

### 3.3 获取前端访问地址

部署成功后，Vercel 会给你一个地址，类似：

```
https://ahst-major-recommend.vercel.app
```

**这就是你分享给同学的最终网址！**

## 第四步：测试验证

### 4.1 功能测试

1. 在浏览器打开你的 Vercel 网址（例如：https://ahst-major-recommend.vercel.app）
2. 测试以下功能：
   - ✓ 页面正常加载，没有报错
   - ✓ 输入高考分数（例如 510）
   - ✓ 选择科目类型（物理类/历史类）
   - ✓ 点击推荐按钮
   - ✓ 看到冲稳保专业列表
   - ✓ 点击专业卡片，看到历年分数线趋势图

### 4.2 常见问题排查

**问题：页面打开但推荐功能报错**

- 检查 Vercel 的环境变量 `VITE_API_BASE_URL` 是否正确设置
- 检查后端是否正常运行（访问 `https://后端地址/api/statistics/`）

**问题：后端部署失败**

- 检查 Render 的日志（Logs 标签页）
- 确认 `build.sh` 文件存在于 `backend/` 目录中
- 确认 `requirements.txt` 文件存在且内容正确

**问题：数据没有导入成功**

- 在 Render 项目的 Shell 中手动运行导入脚本：
```bash
cd backend
python ../database/import_data.py
python ../add_2024_2025_scores.py
```

## 技术支持

如果部署过程中遇到问题：

1. 检查平台日志
   - Vercel: 项目 → Deployment → Build Logs
   - Render: 项目 → Logs

2. 常见解决方案
   - 端口问题：使用环境变量 `$PORT`（Render 自动设置）
   - 数据库权限：SQLite 文件需要写入权限
   - CORS 跨域：Django 的 CORS 配置已允许所有来源

3. 验证步骤
   ```
   ✓ 后端 API 可用
   ✓ 前端页面可访问
   ✓ 前后端通信正常
   ✓ 推荐功能可用
   ✓ 图表可视化正常
   ```
